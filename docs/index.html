Features

Boring infrastructure. Sharp UX.

Everything you'd build yourself if you had three months — already built, tested in production on real properties.

01

Stripe ACH + cards

Tenants pay via bank account (0.8%, capped at $5) or card (2.9% + $0.30). All processed through Stripe. PCI scope: zero.

02

Auto-pay via Subscriptions

Stripe Subscriptions runs the schedule. Your server can be offline at charge time and rent still gets billed. Webhooks reconcile on the way back up.

03

Zitadel OIDC auth

Tenants log in with email + password, passkeys, or federated identity. No password reset emails to handle. No JWT forging vectors.

04

Financial Connections + microdeposits

Instant bank verification via Stripe Financial Connections (Plaid). Fall back to microdeposit verification for banks that need it. Both flows handled.

05

Admin dashboard

Properties, tenants, payment history, debug view. Edit rent and the change syncs to the right Stripe Subscription automatically.

06

Webhook idempotency

Every Stripe event is processed exactly once via a postgres unique-id table. Stripe retries are no-ops. A catchup script replays missed events.

07

Self-hostable

Docker compose for postgres + PostgREST + Caddy. Coolify-friendly. Runs on a $5/mo VPS or your home server. Your tenants' data never touches a third party we manage.

08

Next.js 15 + TypeScript

App Router, React 19, server actions, edge-runtime middleware. Tailwind + shadcn/ui. Code you can actually read and modify.

09

AGPL-3.0

Source available. Modify, deploy, run a portfolio of properties on it. Re-distribute under the same license. No callback to a license server.

Honest fees

Stripe charges what Stripe charges. We add zero.

EZPM itself is free. The only money that moves is what Stripe takes per transaction — same rates you'd get with a direct Stripe integration.

Payment method Stripe rate $1,500 rent fee $2,500 rent fee $5,000 rent fee
Bank account (ACH) 0.8%, capped at $5 $5.00 $5.00 $5.00
Credit / debit card 2.9% + $0.30 $43.80 $72.80 $145.30
EZPM software fee — $0.00 $0.00 $0.00

Settings determine who absorbs the fee. By default tenants see the fee on top of rent; flip a config and the property owner absorbs it instead. Bank account is dramatically cheaper for any rent over $625 (where the $5 cap kicks in).

How it works

From "deploy" to "rent paid" in an afternoon.

You bring a domain, a Stripe account, and a Zitadel instance (or use ours). Done. EZPM does the rest.

1

Deploy with docker compose

Coolify, Render, your home server, anywhere docker runs. One compose file for the app, one for the database stack. ~10 minutes.

2

Connect Stripe + Zitadel

Drop your Stripe keys + webhook secret into env vars. Point Zitadel at the OIDC callback URL. First admin is whoever logs in first.

3

Pre-stage tenants by email

Admin enters tenant email + property + monthly rent + due day. Tenant is invited via Zitadel; they set their own password.

4

Tenant adds a bank or card

Stripe PaymentElement handles the UI. Bank picker for instant ACH verification; manual entry falls through to microdeposits. Subscription auto-creates on first verified PM.

5

Stripe charges on the due day

Every month, automatically. Webhooks update EZPM's payments mirror. Failed charges go into Stripe Smart Retries. Refunds + reversals fire normal webhook events you can see in the admin.

# Bring up the stack
$ git clone github.com/Qureshi-Inc/ezpm
$ cp .env.example .env.local
$ docker compose up -d

# Apply schema
$ psql -f supabase/schema.sql

# Done. Open the admin.
$ open https://rent.example.com/admin

# Next month, automatically:
# invoice.created → payments row inserted
# invoice.payment_succeeded → status=succeeded
# charge.failed (ACH bounce, 1-7d)
# → status=failed, dunning kicks in
Stack

Boring tech, deliberately.

Every piece is something you could read, replace, or fork. Nothing proprietary, nothing magical.

Next.js 15
Framework
App Router, React 19, server actions, edge middleware. TypeScript end-to-end.
Postgres + PostgREST
Data
Self-hosted database with a thin auto-generated REST API. Same shape as Supabase, runs on your hardware.
Stripe
Payments
Customers, Subscriptions, Invoices, Financial Connections. API version pinned, dahlia-compatible.
Zitadel
Auth
OIDC identity provider. Auth.js v5 client. Federated login (Authentik, Google) optional.
Tailwind + shadcn/ui
UI
Familiar component library. Copy any component into your fork and modify directly.
Caddy
Edge
Reverse proxy + automatic TLS. The DB stack uses it to translate Supabase-shaped URLs to plain PostgREST.
Docker compose
Runtime
Two compose files: app + DB stack. Coolify-friendly. Works on any docker host.
AGPL-3.0
License
Source available. Modify and deploy freely. Modifications to derivative SaaS must be open-sourced.
